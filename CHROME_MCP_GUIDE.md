## 概述

Chrome DevTools MCP 是一个强大的浏览器自动化和调试工具集，允许通过自然语言指令控制 Chrome 浏览器，执行各种测试、调试和自动化任务。

---

## 能力分类

### 页面管理

#### 1. 列出所有打开的页面

**功能**: 获取浏览器中所有打开的标签页列表

**提示词示例**:
```
列出浏览器中所有打开的页面
显示当前打开的所有标签页
获取所有浏览器页面列表
```

---

#### 2. 选择页面

**功能**: 选择一个页面作为后续操作的上下文

**提示词示例**:
```
选择第 2 个标签页
切换到 ID 为 1 的页面
激活第一个页面
```

---

#### 3. 打开新页面

**功能**: 在新标签页中打开指定的 URL

**提示词示例**:
```
打开新标签页并访问 https://example.com
在新标签页中打开 https://github.com
创建一个新页面访问 https://localhost:3000
```

---

#### 4. 关闭页面

**功能**: 关闭指定的标签页（最后一个标签页无法关闭）

**提示词示例**:
```
关闭第 2 个标签页
关闭 ID 为 3 的页面
```

---

#### 5. 导航页面

**功能**: 导航到指定 URL、后退、前进或刷新页面

**提示词示例**:
```
导航到 https://example.com
刷新当前页面
后退到上一页
前进到下一页
重新加载页面并忽略缓存
```

---

#### 6. 调整页面大小

**功能**: 调整页面窗口的宽度和高度

**提示词示例**:
```
将页面调整为 1920x1080
设置页面宽度为 1366 高度为 768
调整浏览器窗口大小为移动端尺寸 375x667
```

悬停在下拉菜单上
将鼠标移到这个按钮上
hover 到导航栏的菜单项

#### 11. 拖拽元素

**功能**: 将一个元素拖拽到另一个元素上

**提示词示例**:

拖动滑块到目标位置

```

保存截图为 PNG 格式
```

## 最佳实践

### 1. 工作流程建议

**典型测试流程**:
```
1. 打开新页面或选择现有页面
2. 获取页面快照了解元素结构
3. 与元素交互（点击、填充表单等）
4. 等待特定状态或文本出现
5. 验证结果（截图、检查文本、查看网络请求）
```

**性能分析流程**:
```
1. 开始性能追踪
2. 执行用户操作或页面导航
3. 停止性能追踪
4. 分析性能洞察
5. 运行 Lighthouse 审计获取综合评分
```

### 2. 调试技巧

- 使用 `take_snapshot` 获取元素 uid，再进行精确操作
- 使用 `list_console_messages` 检查是否有 JavaScript 错误
- 使用 `list_network_requests` 验证 API 调用是否成功
- 使用 `evaluate_script` 获取页面内部状态

### 3. 自动化测试建议

- 在操作前先获取快照确认元素存在
- 使用 `wait_for` 等待动态内容加载
- 使用 `emulate` 测试不同设备和网络条件
- 定期截图记录测试过程

### 4. 性能优化建议

- 批量操作使用 `fill_form` 而不是多次 `fill`
- 避免频繁截图，仅在关键步骤截图
- 合理设置 `wait_for` 的超时时间
- 使用性能追踪识别性能瓶颈

## 常见使用场景

### 场景 1: 自动化表单填写和提交

```
1. 打开页面 https://example.com/login
2. 获取页面快照
3. 批量填充表单（用户名、密码）
4. 点击登录按钮
5. 等待 "欢迎" 文本出现
6. 截图验证登录成功
```

### 场景 2: 性能测试

```
1. 导航到目标页面
2. 开始性能追踪
3. 模拟移动设备
4. 模拟慢速网络
5. 执行用户操作
6. 停止性能追踪
7. 运行 Lighthouse 审计
8. 分析性能瓶颈
```

### 场景 3: API 调试

```
1. 打开页面
2. 执行操作触发 API 调用
3. 列出网络请求
4. 获取特定请求的详细信息
5. 检查控制台是否有错误
```

### 场景 4: 响应式测试

```
1. 调整页面到移动端尺寸
2. 截图验证布局
3. 调整到平板尺寸
4. 截图验证布局
5. 调整到桌面尺寸
6. 截图验证布局
```

## 注意事项

1. **元素标识**: 使用 `take_snapshot` 获取元素的 uid，这是最可靠的元素定位方式
2. **异步操作**: 对于动态加载的内容，务必使用 `wait_for` 等待
3. **性能影响**: 性能追踪和内存快照会影响页面性能，仅在需要时使用
4. **网络请求**: 网络请求列表会在页面导航后清空，如需保留请使用 `includePreservedRequests`
5. **最后页面**: 无法关闭最后一个打开的标签页
